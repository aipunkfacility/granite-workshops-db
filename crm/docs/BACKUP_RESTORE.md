# Backup & Restore

## Обзор

Автоматическая система бэкапов защищает данные от потери:
- Бэкап создаётся **перед** каждым изменением файла
- Хранятся последние **10 бэкапов** на файл
- Старые бэкапы удаляются автоматически

## Механизм

### Когда создаётся бэкап

| Операция | Бэкап | Примечание |
|----------|-------|------------|
| `PUT /db/{file}` | ✅ | Перед перезаписью |
| `DELETE /db/{file}` | ✅ | Перед удалением |
| `POST /db/{file}/restore` | ✅ | Перед восстановлением |
| Чтение | ❌ | Не бэкапится |
| Файл не существует | ❌ | Нечего бэкапить |

### Формат имени

```
{original_filename}.{YYYYMMDD}_{HHMMSS}.bak
```

Пример: `Rostov.json.20260403_044857.bak`

### Расположение

```
crm/backups/*.bak
```

## API

### Список всех бэкапов

```bash
curl http://localhost:8000/backups
```

```json
{
  "backups": [
    {"name": "Rostov.json.20260403_000048.bak", "size": 22583, "created": "..."},
    {"name": "Rostov.json.20260402_180000.bak", "size": 22450, "created": "..."}
  ]
}
```

### Бэкапы конкретного файла

```bash
curl http://localhost:8000/db/Rostov.json/backups
```

### Восстановление по имени бэкапа

```bash
curl -X POST http://localhost:8000/restore/Rostov.json.20260403_000048.bak
```

Возвращает содержимое бэкапа:
```json
{
  "success": true,
  "backup": "Rostov.json.20260403_000048.bak",
  "contacts": [...]
}
```

### Автовосстановление (последний бэкап)

```bash
curl -X POST http://localhost:8000/db/Rostov.json/restore
```

Автоматически находит и применяет самый свежий бэкап.

## Ручное восстановление

Если сервер недоступен, восстановите вручную:

```bash
# 1. Найдите бэкап
ls crm/backups/

# 2. Скопируйте вместо текущего файла
copy crm\backups\Rostov.json.20260403_000048.bak crm\db\Rostov.json
```

## Политика хранения

| Параметр | Значение |
|----------|----------|
| Максимум бэкапов на файл | 10 |
| Ротация | По времени (новые → старые) |
| Место хранения | `crm/backups/` |

При превышении лимита старые бэкапы удаляются автоматически.

## Восстановление повреждённого JSON

Если файл повреждён и не читается:

1. Сервер вернёт ошибку 500 с указанием бэкапа:
```json
{"detail": "Invalid JSON: ... Backup available: Rostov.json.20260403_044857.bak"}
```

2. Восстановите:
```bash
curl -X POST http://localhost:8000/db/Rostov.json/restore
```

3. Или вручную:
```bash
copy crm\backups\Rostov.json.20260403_044857.bak crm\db\Rostov.json
```

## Восстановление удалённого файла

1. Найдите бэкап:
```bash
curl http://localhost:8000/backups
```

2. Восстановите содержимое (создаст новый файл):
```bash
curl -X POST http://localhost:8000/restore/Rostov.json.20260403_044857.bak
```

3. Сохраните данные обратно:
```bash
curl -X PUT http://localhost:8000/db/Rostov.json \
  -H "Content-Type: application/json" \
  -d '{"contacts":[...]}'
```

## Ограничения

| Ограничение | Значение |
|-------------|----------|
| Максимальный размер JSON | 10MB (настраивается) |
| Формат бэкапа | Точная копия JSON |
| Шифрование | Нет |

## Логирование

Все операции с бэкапами логируются:

```
crm/logs/crm_server.log
```

Примеры:
```
2026-04-03 04:48:57 - INFO - Created backup: crm/backups/data.json.20260403_044857.bak
2026-04-03 04:49:12 - INFO - Restored from backup: data.json.20260403_044857.bak
```

## Безопасность

- Бэкапы хранятся локально (`crm/backups/`)
- Path traversal защита: только `.bak` файлы
- Файлы не удаляются без создания бэкапа
