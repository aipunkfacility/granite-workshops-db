## Выводы по Scrape-подходу

### Что работает ✅

1. **Быстрый сбор списка** — 10-30 секунд на страницу
2. **Адреса** — извлекаются надёжно
3. **Названия** — извлекаются надёжно
4. **Дешевизна** — минимальный расход credits

### Что НЕ работает ❌

1. **Телефоны** — скрыты за JavaScript-кнопкой
2. **Структура** — нет привязки компания→телефон
3. **Полнота** — только видимый контент

### Решение проблемы телефонов

Вариант A: **Agent для каждой компании**
```bash
firecrawl agent "extract phone and contacts" --urls "..." --timeout 300
```
Минус: дорого, долго (для 35 компаний = ~2 часа)

Вариант B: **Парсить 2GIS/Yell**
```bash
firecrawl scrape "https://2gis.ru/stavropol/firm/..."
```
Плюс: структурированные данные

Вариант C: **Поиск вручную**
```bash
firecrawl search '"{компания}" "{город}" телефон'
```
