# План выполнения задания по поиску мессенджеров для компаний

## Этап 1: Подготовка и анализ данных
1.1. Загрузка исходного CSV файла `rostov_enriched.csv`
1.2. Идентификация компаний для обработки:
   - Исключение 10 компаний без сайта (CMP0023, CMP0031, CMP0062, CMP0065, CMP0084, CMP0094, CMP0097, CMP0098, CMP0107, CMP0108)
   - Исключение 1 компании с уже заполненными всеми мессенджерами (CMP0026)
   - Выделение 56 компаний с сайтами, требующих обработки (53 уникальных домена)
1.3. Создание структуры данных для отслеживания прогресса обработки
1.4. Подготовка списка исключений и специальных случаев (сайты с несколькими компаниями)

## Этап 2: Разработка алгоритма поиска мессенджеров
2.1. Определение последовательности проверки для каждого сайта:
   - Главная страница (шапка, футер, плавающие элементы)
   - Страница "Контакты" и варианты URLs
   - Страница "О нас" и варианты URLs
   - Футер всех доступных страниц
2.2. Определение паттернов поиска для каждого мессенджера:
   - VK: иконки, ссылки vk.com/..., кнопки "Мы ВКонтакте", проверка принадлежности группы компании
   - WhatsApp: иконки, ссылки wa.me/..., номера с подписью WhatsApp, конвертация форматов телефонных номеров
   - Telegram: иконки, ссылки t.me/..., кнопки "Написать в Telegram", проверка официальности канала/бота
2.3. Определение fallback-стратегии (веб-поиск):
   - Запросы вида "[Название] [Город] VK"
   - Запросы вида "[Название] [Город] Telegram"
   - Запросы вида "[Название] [Город] WhatsApp"
   - Валидация найденных результатов на принадлежность компании

## Этап 3: Реализация парсинга и извлечения данных
3.1. Настройка HTTP клиента с обработкой ошибок и таймаутов
3.2. Реализация функций извлечения ссылок:
   - Поиск по атрибутам href и содержимому тегов
   - Поиск по классам и id, связанным с соцсетями
   - Извлечение текста вокруг найденных элементов для валидации
3.3. Реализация обработкиrelative и absolute URL
3.4. Реализация конвертации телефонных номеров:
   - Удаление +7, скобок, пробелов, дефисов
   - Приведение к формату 7XXXXXXXXXXX
   - Проверка длины (11 цифр)
3.5. Реализация проверки принадлежности найденных групп компании:
   - Сравнение названия, телефона, адреса из информации группы с данными компании
   - Отклонение явных личных профилей сотрудников (если нет подтверждения официальности)

## Этап 4: Обработка специальных случаев
4.1. Обработка сайтов с несколькими компаниями на одном домене:
   - pamyatnik-doneck.ru (CMP0034, CMP0035, CMP0039)
   - granite161.ru (CMP0032)
   - taganrog-pamyatniki.ru (CMP0099, CMP0100)
   - Унифицированный подход к поиску для всех компаний на домене
4.2. Обработка множественных WhatsApp номеров:
   - Сбор всех найденных номеров
   - Форматирование каждого в требуемый вид
   - Объединение через запятую
4.3. Предотвращение конфликтов WhatsApp/Telegram:
   - Если номер привязан к WhatsApp на сайте - не записывать его в Telegram
   - Исключение случаев, когда на сайте явно указано и то, и другое с разными номерами

## Этап 5: Веб-поиск и валидация результатов
5.1. Интеграция с поисковым API для fallback-поиска
5.2. Реализация логики выбора лучшего результата:
   - Приоритет официальных групп/каналов
   - Проверка соответствия названия и ubicación
   - Отклонение явных нерелевантных результатов
5.3. Валидация формата ссылок перед записью:
   - VK: обязательно https://vk.com/...
   - Telegram: обязательно https://t.me/... (без @)
   - WhatsApp: обязательно https://wa.me/7XXXXXXXXXXX
5.4. Логирование всех действий для отладки и аудита

## Этап 6: Генерация отчетов и итоговых файлов
6.1. Создание обновленного CSV файла `rostov_enriched_2.csv`:
   - Копирование всех неизмененных столбцов из исходного файла
   - Заполнение только пустых полей VK, Telegram, WhatsApp
   - Сохранение уже заполненных значений без изменений
6.2. Формирование детального отчета:
   - Статистика по обработанным/недоступным сайтам
   - По каждой компании: найденные мессенджеры и источник (главная, контакты, футер, виджет, поиск)
   - Перечень компаний, для которых ничего не найдено
   - Проблемные случаи с пояснениями
6.3. Проверка соответствия требований:
   - Точное количество строк (67)
   - Неизмененность защищенных столбцов
   - Корректность формата всех ссылок
   - Отсутствие выдуманных данных

## Этап 7: Финальная валидация
7.1. Сверка итогового файла с исходным по неизменяемым полям
7.2. Проверка, что не удалены и не пропущены какие-либо компании
7.3. Валидация формата всех заполненных ссылок через регулярные выражения
7.4. Подготовка рекомендаций по ручной проверке сомнительных случаев
7.5. Финальное подтверждение готовности результата к использованию
