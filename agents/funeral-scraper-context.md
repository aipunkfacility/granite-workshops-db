# Funeral Agency Scraper — Context

## Структура проекта

```
funeral-agency-db/
├── config.yaml              # 44 города России 500К+
├── sources/
│   └── directories.md       # Источники для поиска
├── agents/
│   └── funeral-scraper.md   # Агент парсинга
├── scripts/
│   ├── scrape_city.py      # Python скрипт сбора
│   └── pyproject.toml     # Зависимости
├── cities/                  # Данные по городам
│   └── {city-name}/
│       ├── companies.md     # ВСЕ компании (база)
│       ├── companies.json  # JSON с данными
│       ├── data.md         # Только с Telegram (приоритет)
│       ├── summary.md       # Саммари
│       └── report.md       # Отчёт с рекомендациями
└── reports/
    └── progress.md          # Трекер прогресса
```

## Workflow (3 этапа)

### Этап 1: Сбор базы → `companies.md`
Собрать ВСЕ организации без фильтрации из источников.

### Этап 2: Обогащение Telegram
Для каждого номера: поиск `"{номер}" Telegram` → добавить username.

### Этап 3: Приоритизация
- С Telegram → `data.md`, `summary.md`, `report.md`
- Без Telegram → остаются в `companies.md`

## Команды

- **Начать город:** `Начни с города [название]`
- **Показать прогресс:** `Покажи прогресс`
- **Следующий город:** `Следующий — [город]`

## Что искать

### Контакты (обязательно)
- Телефон, адрес, сайт
- Email (искать на сайте!)

### Соцсети и мессенджеры (важно!)
- VK, Telegram, WhatsApp
- Поиск по номеру: `"{номер}" Telegram`

### Дополнительно
- Станки (лазерные/ударные)
- Услуги (ретушь, восстановление фото)

## Источники

1. **Карты:** 2GIS, Яндекс.Карты, Google Maps
2. **Каталоги:** Yell, JSprav, Firmsru
3. **Доски объявлений:** Авито, Юла

## Важно

- Собирать ВСЕ организации (цель — сотни)
- Для каждого номера искать Telegram
- Не придумывать данные
- Проверять актуальность ссылок
